# Sim-U-Kraft 新版重写当前进度与后续计划

本文是后续进度与计划的统一维护文件。接口调用规范见 [接口用法.md](接口用法.md)。

## 项目定位

新版 Sim-U-Kraft 以旧版玩法体验为规则来源，但底层按 NeoForge 1.21.1、LDLib2、SQLite 和服务层架构重新实现。

核心原则：

- 服务端是权威端，客户端只负责显示和请求。
- 长期数据进入 SQLite，运行时缓存必须有作用域和失效策略。
- 旧版规则优先还原，旧版低性能实现不直接继承。
- NPC、建筑、城市、经济、材料和寻路都通过 Service 串联，避免 GUI 或 Packet 写业务。
- 任何新功能都要考虑多人协作、存档安全、服务器性能和后续扩展。

## 最近更新（2026-06-13）

- 修复 `CityManager.loadFromSqlite` 致命 bug：原实现在调用 `loadCities` **之前**就把 `sqliteLoaded` 置为 true；崩溃重启时若 SQLite 连接失败（文件锁、WAL 未恢复等），Manager 会静默持有空城市数据且永不重试，导致玩家城市权限/成员关系全部丢失（城市掉权）。现改为仅在 `loadCities` 成功返回非 null 后才置位，与 `CitizenManager` 保持一致。
- 将全局周期性保存间隔从 1200 ticks（约60秒）缩短为 200 ticks（约10秒）：`SimuKraft.java` 中 `saveDimensionSqlite` 与 `saveGlobalSqlite` 的触发条件同步修改，崩溃时最多丢失约10秒数据。SQLite WAL 模式下此写入频率无明显性能影响。
- 确认 `/simukraft reload database` 指令已完整落地：9 个 Manager（`CityManager`、`CityChunkManager`、`CityPoiManager`、`CitizenManager`、`CommercialBoxManager`、`CommercialStockManager`、`IndustrialBoxManager`、`FarmlandBoxManager`、`LogisticsManager`）均已有 `reloadFromSqlite(ServerLevel)`，`SimuKraftCommand` 已注册 `reload database` 子命令，管理员可在不重启服务器的情况下全量重载内存数据。

## 最近更新（2026-06-10）

- 修复玩家地面投喂链路：`CitizenDroppedFoodService` 会让未吃饱的已加载 NPC 分散扫描 2 格内可食用掉落物，消耗一份食物后同步饱食度、手持食物表现、吃饭声音和挥手动画。工作中的 NPC 只保护无主掉落物；玩家主动丢出的食物仍视为投喂。
- 新增 `CitizenFoodConsumptionService` 统一处理食物属性读取、饱食度增加、吃饭表现和临时手持物恢复，避免地面投喂、自动买饭各自复制一套食物逻辑。
- 饱食度已从 `CitizenData` 和 SQLite 迁出：唯一持久化来源为 `CitizenEntity` 的 `Hunger` NBT，默认值为 `CitizenEntity.DEFAULT_HUNGER`；`SimuSqliteSchema` 会删除旧 `citizens.hunger` 列，`CitizenData` 不再读写 `Hunger` 标签。
- NPC 饥饿下降只作用于已加载实体；实体未加载时不再通过数据库模拟饱食度，雇佣列表等无实体场景使用默认满值兜底，避免把运行时实体状态重新写回长期市民档案。
- LDLib 文本框已加入 IMBlocker 软兼容：`MixinLdlibTextField` 为文本框焦点、失焦、字符探测和光标移动接入 `LdlibTextFieldImeCompat` 动态代理，未安装 IMBlocker 时不会产生硬依赖，用于修复中文输入框冲突。
- 参考旧版接入 Xaero World Map：`SimuKraftCityHighlighter` 从 `ClientCityChunkCache` 读取城市区块，在 Xaero 地图中渲染城市填充色、外边界和悬停城市名；`MixinWorldMapSession` 在地图会话初始化后注册高亮器，并使用 `@Pseudo` 保持 Xaero 未安装时的软兼容。
- 同步语言文件新增 `gui.simukraft.xaero.city.current` 与 `gui.simukraft.xaero.city.other`，并通过 `testClasses` 覆盖了 `CitizenData` 不再持久化 `Hunger` 的测试断言。

## 最近更新（2026-06-09）

- 模组配置 GUI 完成高 UI 缩放兼容：公共 `SimuKraftConfigWidgets` 统一按 `Minecraft.getInstance().getWindow().getGuiScaledWidth/Height()` 计算可用逻辑尺寸，选择页、客户端配置页、服务端配置页和材料配置独立页不再依赖固定窗口尺寸硬撑屏幕。
- 服务端配置页的 Tab 宽度和页脚高度改为按窗口尺寸计算；页脚按钮支持收缩和换行，输入框、下拉框也允许在窄屏下收缩，避免高缩放或小窗口时互相挤压。
- 材料配置独立页顶部已压缩：外层标题栏改为紧凑高度，材料列表/分类组编辑器使用紧凑分组标题、较低说明文本和搜索区高度，给材料列表留出更多可视空间。
- 材料分类组编辑器在窄屏下从三栏分割布局切换为上下堆叠滚动布局，保证所有分栏在高缩放下仍可操作。
- 本次修改只影响客户端 LDLib2/Taffy 布局，不改配置数据、网络包、SQLite 或服务端规则；配置项未变更，因此不需要同步语言文件或 NeoForge 配置项定义。

## 最近更新（2026-06-06）

- 修复 NPC 自动买饭后的头顶状态残留：`去商店买饭`、`正在买饭`、`太饿了，等待商店供餐` 现在都作为临时流程状态处理，吃饱后不会被恢复成持久状态，避免出现“吃的饱饱的”和“太饿了”同时显示。
- 自动买饭流程新增 `正在吃饭` 状态，并在买到食物后先同步满饥饿值、手持食物和吃饭动画；流程结束后恢复原本工作状态，只清理买饭相关临时详情。
- 增加已吃饱 NPC 的脏状态自修复逻辑：如果旧存档或异常流程里残留了买饭状态，服务端下一次自喂食 tick 会清理状态标签并同步实体。
- 修复 NPC 梯子移动：爬梯和下梯都使用统一的垂直辅助速度，不再只会上梯不会下梯。
- 梯子出口横移增加头部碰撞检查：NPC 只有在当前头部空间和朝下一段横移方向探测的头部空间都无碰撞时，才会从梯子向四周移动，减少梯口卡头和提前横移。
- 本次修改保持服务端为权威端，客户端只读取实体同步数据渲染头顶文本；语言文件已补充 `gui.npc.status.eating_food`。

## 最近更新（2026-06-05）

- 商业系统第一阶段已落地：交易项改为 `cost -> result`，资源统一支持资金和物品，`visibleTo` 控制玩家/NPC/混合可见，支持以物换物。
- 玩家右键商业员工 NPC 会打开商业交易界面；非商业员工仍打开 NPC 信息界面。
- 商业库存支持最大库存、初始库存、补货数量和补货间隔；补货按 `level.getGameTime()` 计算，不受 `/time set` 影响。
- 建筑资源已参考旧版 `_files.txt` 列表法内置并自动复制到游戏根目录 `simukraftbuilding`，现有自定义文件不会被覆盖。
- 住宅租金已改为按服务器运行 tick 推导结算日，修复第二天不收租的问题。
- NPC 转向/拐角跟随不再额外降低移动速度，减少转向阶段明显顿挫。
- 已修复高危雇佣越权：雇佣列表、雇佣分配、解雇都会校验岗位来源、城市管理权限和 NPC 城市归属。
- 已修复建造/规划运行时缓存清理前缀大小写不一致，避免 Windows 存档路径大小写导致缓存串档或残留。

## 当前已完成

### 基础工程

- NeoForge 1.21.1 工程可编译运行。
- Java 21、Gradle、资源处理链路可用。
- LDLib2 已接入，当前依赖为 CurseMaven 版本。
- 主类、注册表、方块、实体、声音、网络包基础结构已形成。
- SQLite 主数据库和建筑结构数据库已接入。

### 城市系统

- 城市核心方块可作为管理入口。
- 支持创建城市、查询城市、成员权限、城市资金、财政流水。
- 支持城市区块购买、相邻校验、资金扣除、失败回滚。
- 客户端城市地图、区块边界、城市核心标记和基础交互已可用。
- 已按旧版思路联动 Xaero World Map：客户端同步到的城市区块会在 Xaero 地图中显示填充、外边界和城市名称提示。
- 城市 POI 已作为住宅、岗位、建筑生态的基础数据来源。

### 建筑系统

- 建筑盒、建筑资源目录、建筑列表和建筑预览已可用。
- 已按旧版 `_files.txt` 列表法内置建筑资源，`residential/commercial/industry/public/other` 五类建筑会随资源打包；运行时扫描建筑列表前自动把缺失文件复制到游戏根目录 `simukraftbuilding/<分类>`。
- 服务端 `BuildingCatalog` 与客户端建筑列表缓存都会在读取前准备内置资源，保留玩家已经修改或新增的根目录建筑文件。
- 支持建筑旋转、预览移动、边界渲染和空间占用判断。
- 支持建造任务、建造进度、材料消耗、完成登记。
- 建筑完成后可登记已放置建筑、住宅床位 POI、完成音效和弹窗消息。
- 住宅控制盒可以显示建筑、居民、容量、床位和边界。
- 建筑拆除链路已具备基础清理能力。
- 规划师职业(`PlannerWorkService`)已可用：在建筑盒雇佣规划师，开"规划区域"界面(键盘操作自由视角、3D 框选、实时线框预览、O 切换操作)选清除/填充/替换并框选区域；确认后服务端按体积一次性预扣城市资金(可在模组 GUI 配置页调整，默认清除/填充 0.02、替换 0.04)，由规划师在建筑盒处逐块执行。填充/替换的真实方块取自紧贴建筑盒的箱子(主手=填充/目标、副手=替换源)，清除掉落物入该箱；夜间随建造休息窗口暂停；任务进 `planning_tasks` 表可恢复；基岩/建筑盒/容器受保护不破坏。
- 已注册 NeoForge 配置屏(`IConfigScreenFactory`)，模组列表"模拟大都市"的配置按钮可打开 GUI 配置页(含规划师计费、寻路、材料、建造等所有服务端配置)。

### 材料系统

- 已有通用材料策略、材料请求、材料结果、材料缓存和通用容器访问。
- 建造师可从紧贴工作方块的箱子读取材料。
- 支持基础模式、专家模式、可替代材料组和跳过列表。
- 缺材料提示已改为右侧滑出弹窗，支持合并重复消息。
- 材料弹窗支持物品图标和长文本自适应高度。
- 材料配置 GUI 已支持高 UI 缩放下的窗口尺寸夹取、顶部紧凑显示和窄屏分类组堆叠布局。

### NPC 系统

- 市民实体、持久化数据、姓名、性别、年龄、寿命、健康、幸福、技能等长期字段已具备；饱食度独立保存在实体 `Hunger` NBT，不再进入 `CitizenData` 或 SQLite。
- 支持城市绑定、住宅绑定、工作点绑定、职业状态和雇佣/解雇。
- NPC 死亡后保留数据并登记死亡，不再直接删除记录。
- 新 NPC 加入住宅时优先生成在床边安全位置。
- 支持夜间回家、第二天返回工作岗位、无任务建筑师回岗位待命。
- 支持职业统一手持物和动作表现，建筑师有任务时右手持圆石并挥动。
- 已恢复饥饿下降、自动买饭和玩家地面投喂；食物效果统一使用原版/模组 `FoodProperties`，吃饭只更新实体 NBT 并同步表现。

### 寻路系统

- 已有 `CitizenNavigationService` 统一入口。
- 支持近距离真实寻路、远距离按配置传送兜底。
- 支持行走、奔跑、跳跃、游泳、梯子/藤蔓/脚手架攀爬。
- 主线程采集快照，异步只处理不可变快照，不直接访问世界。
- 支持路径缓存、快照缓存、限流、失败冷却和卡住传送。
- 支持木门开门；不主动开铁门、活板门、栅栏门。
- 支持 Alt+P 客户端调试渲染和寻路测试/压力测试指令。
- 已加入拥堵协调和局部让行逻辑，仍需继续打磨拐角和身位校准。
- 转向/拐角跟随不再额外降低移动速度，速度仍由移动意图和移动模式统一计算。

### UI 与提示

- 主要 GUI 使用 LDLib2。
- 城市核心和 NPC 信息界面已改为普通窗口框架，不再继承 LDLib2 `Editor`，避免触发 UI Scale 自动设置。
- 已新增 `SimuKraftWindowFrame` 作为普通窗口容器。
- 模组配置界面已统一按当前 GUI 缩放后的逻辑分辨率计算窗口尺寸；服务端配置页 Tab、页脚按钮、输入框和下拉框支持窄屏收缩/换行。
- LDLib 文本框已通过 IMBlocker 软兼容支持中文输入，焦点代理和光标位置只在客户端处理，不影响服务端规则。
- 已新增右侧滑出弹窗，用于系统消息、材料提醒、金钱提示和建筑完成提示。
- 弹窗支持重复消息合并、Logo 显示、物品图标和高度自适应。

### 农田系统

- 农田盒（`nsuk_farmland_box`）已可作为农场控制入口，右键打开 LDLib2 普通窗口。
- 配置进 SQLite（`farmland_boxes` 表）+ SavedData 兜底，含作物、作业区域、运行状态。
- 复用既有系统：放置时自动登记 `FARMLAND` POI 并限制在城内，破坏时注销；农民经既有雇佣链以 `workplacePos=箱子位置` 绑定。
- `FarmlandFarmingService` 实现真种真收：移动农民→耕地浇水→消耗箱内真实种子种植→等原版自然生长→收获成熟作物真实掉落物入箱（满则掉地），绝不凭空产出。
- 支持小麦、胡萝卜、马铃薯、甜菜根（满铺）和西瓜、南瓜（藤蔓棋盘）。
- 设置区域为 `FarmlandAreaScreen`（键盘操作的自由视角界面，实现 `FreeCameraScreen` 标记接口）：打开即进入自由视角可飞行观察，不暗化世界、实时线框预览（复用 `BuildingBoundsRenderer`）；面向相对模型（长=纵深/宽=横向/离盒距离/左右平移），默认以玩家朝向、农田盒外一格为近边底线，不再以盒子为中心；确认/取消后退出自由视角、回到玩家视角，自定义 min/max 发服务端校验（限尺寸、防重叠）。键鼠 Mixin 的自由视角白名单已改为按 `FreeCameraScreen` 接口判断。
- 区域界面 ESC/取消会丢弃未保存选区（不发设置包并清除候选线框）；自由视角飞行 R 可把区域对齐到当前相机朝向。
- 视线对准农田盒持续超过 1 秒，`FarmlandHoverPreview` 会请求并显示该盒已保存作业区域的线框（轻量请求/响应包，多人安全，仅展示）。
- 切换作物为弹出菜单 `FarmlandCropScreen`：列出全部作物并标注当前选中，点选后发服务端设置。
- 仓储箱不再手动绑定，开始耕作及每个工作周期由 `FarmlandBoxService.resolveAdjacentChest` 自动检测紧贴农田盒六个面的容器，其它位置不考虑；无相邻箱时不能开始/暂停作业。
- 性能：只处理运行中的盒子，每盒独立节流，游标分片，不全量扫描区域，未加载区块跳过。

### 商业系统

- 已新增商业控制箱、商业员工岗位、商业箱管理界面、商业交易菜单、网络包和 SQLite 持久化表。
- 商业定义加载器会优先读取建筑 `.sk` 旁边同名 `.json`，并支持内置 `assets/simukraft/commercial/<id>.json` 作为兜底。
- 新交易配置不再区分建筑级店铺模式，交易项使用 `visibleTo=player/npc/mixed` 决定玩家、NPC 或双方可见。
- `cost` 和 `result` 统一为资源列表，资源支持 `{money}` 与 `{item,count}`，因此资金换物、物品换资金、物品换物品走同一套服务端逻辑。
- 交易执行统一在服务端验证可见性、玩家物品、城市资金、库存上下限、结果投递空间和服务端报价，防止客户端改价。
- 库存按 `stock.item` 明确绑定交易影响的物品，物品进入商店增加库存，物品离开商店减少库存；资金进入/离开城市账本并记录财政流水。
- 补货按 `lastRestockGameTime` 与当前 `level.getGameTime()` 差值计算，跨多个间隔会一次性结算，不受 `/time set` 影响。
- 玩家右键商业员工 NPC 打开仿村民交易风格界面：左侧报价列表，中间成本到产出，包含出售/收购/以物换物分类、搜索、库存状态和交易确认。
- NPC 自动经营只处理 `visibleTo=npc/mixed` 的交易项，商业箱拆除时会停止营业、释放员工并删除商业箱和库存 SQLite 记录。

### 经济与 HUD

- 城市资金、财政流水、HUD 同步已具备。
- 住宅租金按 `level.getGameTime()` 推导运行日结算，第二天会按城市聚合入账并写入财政流水，避免 `/time set` 干扰收租窗口。
- HUD 显示城市、资金、人口、日期等基础信息。

## 当前主要风险

- NPC 在窄口、墙角、半格碰撞体附近仍需要实测通过率；转向不再降速后，重点观察 waypoint 到达判定和局部目标是否会造成贴边。
- 寻路调试线正确不代表原版 `PathNavigation` 一定严格按线行走，需要继续限制局部目标和转向时机。
- 材料系统必须持续避免每 tick 扫箱子，后续新增职业工作也要复用缓存。
- 城市核心、NPC 信息等非配置 LDLib 窗口仍需继续实测是否完全不会改动玩家 UI Scale；配置界面已完成高缩放兼容，但仍建议做游戏内多倍率回归。
- LDLib 文本框中文输入依赖 IMBlocker 的运行时反射接口，已做软失败保护；如果 IMBlocker 后续改包名或接口名，需要优先检查 `LdlibTextFieldImeCompat`。
- Xaero 城市区块高亮依赖 `ClientCityChunkCache` 已收到当前维度数据，以及 Xaero World Map 的 highlighter 注册点未改名；若地图不显示，优先排查区块同步、`MixinWorldMapSession` 应用和 Xaero 高亮器字段名。
- NPC 饱食度只随已加载实体下降和保存；未加载市民不会在 SQLite 中离线扣饱食度，这是为了避免实体 NBT 与数据库双写冲突。
- 建筑拆除、床位、居民绑定、工作点绑定之间还要加强一致性修复。
- 数据库 schema 后续扩展前要规划版本字段和迁移策略。
- 内置建筑资源当前只复制缺失文件，不覆盖玩家自定义；后续若内置建筑更新，需要设计版本化刷新策略，避免误覆盖玩家改造。
- 旧版商业建筑 JSON 已可作为同名配置入口；内置商业 `offers` 的资金价格已按旧版 `retail` 语义回归，零售商品按单个价格，建材店和伐木工小屋等批发商品按一组 64 个的组价处理。

## 已知代码问题清单（扫描结果，待修）

一次代码扫描发现的具体缺陷，按严重度排列。✅=已对照源码确认，⚠️=待复核。

高危（已修复）：

- ✅ 已修复：雇佣越权。`NpcHireAssignPacket` / `NpcHireListRequestPacket` / `NpcHireFirePacket` 统一走 `NpcHireAccessValidator`，服务端校验来源方块、城市管理权限、NPC 同城归属、岗位占用和解雇目标绑定。
- ✅ 已修复：缓存清理大小写不匹配。`BuilderConstructionService.clearServerCaches` 与同模式的 `PlannerWorkService.clearServerCaches` 均使用小写 server key 匹配小写 runtime key，避免 Windows 盘符大小写导致缓存清不掉。

中危：

- ✅ 全量实体扫描：`CitizenTeleportService.findCitizenEntity`(:74) 每次查实体都 `getAllEntities()` 全量遍历，且在查找方法里顺手 discard 重复实体（副作用）；被寻路/雇佣高频调用，O(实体数)/次。
- ✅ 全量实体扫描：`CitizenNavigationService` 的 `countLoadedCitizens`(:413)/`findNearest`(:436) 在 `requestMove` 准入控制里再次 `getAllEntities()` 全量扫描。
- ✅ 建城无回滚：`CityCoreCreateCityPacket:81-86`，`createCity` 落库后若 `assignInitialArea` 失败直接 return 不回滚，玩家得到无领地城市且被 `findPlayerCity` 非空永久锁定无法再建城。
- ✅ 拆除丢物：`PlacedBuildingDemolitionService:30` 用 `setBlock(AIR, 3)` 不 `dropResources`，建筑内箱子/熔炉等内容物全部丢失。
- ✅ 批量存档 no-op：`CitizenManager.saveToSqlite`(:53-57) 方法体为空、`save()` 返回空 tag，市民完全依赖增量写；崩溃时未增量落库的字段会丢（需复核增量是否覆盖全部字段）。
- ⚠️ 收租占用判定不一致：`ResidentialRentService` 按整建筑计费，但占用用 `homeId→POI` 与 `getPoiAt` 两套判定；多床建筑只要 1 床有人就收整建筑租金，POI 替换后可能漏收。
- ⚠️ 人口增长限流失效：`PopulationGrowthService.tick:22-28`，`maxPerInterval` 只限生成、不限 `fillVacantHomes` 全量重分配，每间隔对每城市全量扫描市民。
- ⚠️ 入住空转：`CitizenHousingService.spawnCitizensForVacantHomes`，`ensureCitizen` 返回 null 时不 break，可能反复刷实体；依赖 `setHome` 即时反映到 `vacantHomes` 查询。

低危：

- ⚠️ `CityPoiManager.replacePoi:217`/`releaseCity:165`：`posIndex.remove` 用非 immutable 的 `pos()`，插入用 `.immutable()`，极端情况下残留 pos→id 映射。
- ⚠️ `CitizenManager.getOrCreate:138`：`aiQueue.contains()` 对 `ConcurrentLinkedQueue` 是 O(n) 线性扫描，且可能重复入队。
- ⚠️ `ResidentialRentService:159`：同一玩家待发收入提示用 `put` 覆盖，时间跳跃时早一条被静默丢弃（仅显示偏差）。
