## 一、项目当前阶段

当前项目已经完成“基础架构与核心城市系统”的重建，正在从“底层可用”推进到“玩法闭环可玩”。

可以把当前阶段理解为：

```text
基础设施已成型。
核心数据链已稳定。
主要问题从存储与架构，转向玩法系统与行为系统。
```

当前底层已经明确：

- 服务端是权威方。
- 客户端只负责展示与交互请求。
- SQLite 是长期持久化层。
- SavedData 只保留世界生命周期桥接和运行期管理器入口。
- 高频数据优先增量保存，不再依赖整块全量重写。
- NPC 职业不会自动分配，必须由玩家主动雇佣。


### 12. 最近已修复的关键问题

已完成：

- builder / planner 被错误映射为其他职业的问题。
- 工作状态被普通 status 覆盖的问题。
- 雇佣后 NPC 过一会儿回退为空闲的问题。
- 重进后实体与 SQLite 冲突时，旧 data 覆盖实体正确职业的问题。
- 当前已改成：
  - 若实体已有正确职业和工作状态，而 data 仍是空状态
  - 则优先使用实体修复 data
  - 并立即写回 SQLite

这意味着：

```text
重进后不会再轻易因为旧 data 把 NPC 覆盖回无业/空闲。
```

### 1. 城市核心雇佣管理页

当前虽然已有雇佣底层和建筑盒入口，但正式的城市核心雇佣管理页还没有完整落地。

目标：

- 查看空闲 NPC。
- 查看岗位来源和岗位容量。
- 查看已雇佣 NPC。
- 在城市核心中完成雇佣与解雇。

这是接下来最值得优先完成的一块，因为它会把当前岗位系统、NPC 数据和雇佣系统正式串成玩法闭环。

### 2. NPC 家与工作点可视化

目标：

- 显示 NPC 住在哪里。
- 显示 NPC 在哪里工作。
- 让玩家能看到住宅和工作点绑定了哪些 NPC。

这是后续工作 AI、休息系统和行为调试的前置条件。

### 3. 旧版饥饿系统复刻

当前已有饥饿字段和状态展示，但行为层还没完整恢复。

目标：

- 饥饿持续下降。
- 饥饿等级影响行为。
- NPC 会买食物、吃食物、恢复饥饿值。
- 规则尽量参考旧版，不主动改变玩法。

### 4. 旧版休息系统复刻

目标：

- NPC 回家休息。
- 休息中状态。
- 休息结束后恢复空闲或工作。

### 5. NPC 工作 AI

目标：

- 农民去农田。
- 商业员工去商业 POI。
- 工业员工去工业 POI。
- 物流员工去物流节点。
- 守卫去防御点。

规则保持：

```text
AI 只执行已有职业。
AI 不会自动获得职业。
AI 不会自动抢岗位。
```

## 七、当前必须继续坚持的原则

1. 不恢复玩家存款、取款系统。
2. 不允许 NPC 自动获得职业。
3. 不允许 AI 自行更换职业。
4. 客户端不能直接决定资金、区块、雇佣和物品变化。
5. 旧版能查到规则时优先按旧版还原。
6. 性能优化不能以改变玩家体验为代价。
7. 城市核心是管理终端，不是城市生态 POI。
8. 城市、POI、职业、地图、NPC 系统必须继续依赖索引和缓存，避免全量扫描。
9. 新增长期数据时优先建 SQLite 表和索引。
10. 增量保存入口必须继续集中在管理器或服务层，GUI 和网络包不直接写数据库。
